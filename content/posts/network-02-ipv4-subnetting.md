> **개요** — IP 주소는 호스트와 네트워크 장비를 식별하는 **논리 주소**다. 이 글은 IPv4 표기법, 클래스 체계, 서브네팅과 CIDR, 공인/사설 IP, 그리고 NAT 동작을 정리한다.

---

## 1. IP Address 개요

- L4에서 데이터를 캡슐화한 후, **L3에서 IP 헤더(Src IP / Dest IP)가 추가됨**
- **수정·설정 가능한 논리 주소** (MAC과 달리 물리적으로 고정되지 않음)
- 호스트와 네트워크 장비를 식별
- 인터넷(WAN)상 컴퓨터·라우터는 고유한 IP 주소를 할당받고, IP 주소를 이용해 통신
- **인터넷상의 공인 IP는 전세계적으로 고유** (중복 불가)
- IPv4: 주력 (약 43억 개, 2³² ≈ 4.29 × 10⁹)
- IPv6: 대체 (2¹²⁸ ≈ 3.4 × 10³⁸개 — 사실상 무한에 가까움)

---

## 2. IPv4 주소 표기

- **32bit**, 8bit(옥텟) 4그룹으로 나눠 `.`으로 구분, 각 옥텟을 10진수로 표시
- 범위: `0.0.0.0` ~ `255.255.255.255`
- 예: `192.168.0.4`

```
192       . 168       . 0         . 4
11000000  . 10101000  . 00000000  . 00000100
```

---

## 3. Network Part + Host Part

호스트가 속한 Network를 구분하기 위해 IP 주소는 두 부분으로 나뉜다.

| 부분 | 역할 | 비유 |
| --- | --- | --- |
| **Network Part** | Network 식별 | 아파트 동 |
| **Host Part** | Network 내의 Host 식별 | 아파트 호수 |

> LAN은 스위치로 구성되며, **1 LAN = 1 Network**
> - 동일 네트워크 = IP의 Network Part가 동일 → LAN, Switch 통신(L2), MAC 기반
> - 서로 다른 네트워크 = IP의 Network Part가 다름 → WAN, Router 통신(L3), IP 기반

---

## 4. 클래스 (Classful Addressing)

> 과거에는 IP 주소를 5개 클래스로 나눠 사용했다. 현재는 CIDR(Classless)로 대체되었지만 개념상 알아둘 필요가 있다.

| 클래스 | 1옥텟 범위 | Network/Host 비트 | 용도 |
| --- | --- | --- | --- |
| **A** | 1 ~ 126 | 8 / 24 | 대규모 네트워크 |
| **B** | 128 ~ 191 | 16 / 16 | 중규모 네트워크 |
| **C** | 192 ~ 223 | 24 / 8 | 소규모 네트워크 |
| **D** | 224 ~ 239 | - | **Multicast 전용** |
| **E** | 240 ~ 255 | - | **실험/연구 용도** |

### 예약된 특수 대역

| 대역 | 용도 |
| --- | --- |
| `127.0.0.0/8` | **Loopback** (자기 자신, localhost) |
| `0.0.0.0/8` | 현재 네트워크 (예약) |
| `169.254.0.0/16` | Link-local (DHCP 실패 시 자동 할당) |
| `224.0.0.0/4` | Multicast |
| `255.255.255.255` | Limited Broadcast |

## 5. Subnetting (Classful → Classless)

> Classful 주소를 그대로 사용하면 IP 낭비가 심하다. 필요한 IP 수만큼 기존 Network를 분할하여 사용하는 것이 **서브네팅**.

- **Network Part 확장** / **Host Part 축소**
- 분할된 Subnet은 서로 **다른 네트워크**로 동작 (L3 통신 필요)

### Subnet 내 예약 주소

| 주소 | 의미 |
| --- | --- |
| Host 부분이 **모두 0** | Network 주소 (호스트 할당 불가) |
| Host 부분이 **모두 1** | Broadcast 주소 (호스트 할당 불가) |

예: `192.168.1.0/24` 서브넷
- Network 주소: `192.168.1.0`
- Broadcast 주소: `192.168.1.255`
- 사용 가능 호스트: `192.168.1.1` ~ `192.168.1.254` (254개)

---

## 6. Subnet Mask & CIDR 표기

- Subnetting으로 나뉜 Network Part를 구분하기 위해 사용
- 32bit, 8bit(옥텟) 4그룹으로 표기
- IP 주소의 **Network 부분을 `1`**, **Host 부분을 `0`** 으로 표시
- 반드시 **연속된 1**, 그 뒤 **연속된 0** 구조 (중간에 섞이지 않음)

### CIDR 표기

- `IP/n` 형식. `n` = Network 부분의 비트 수
- 예: `192.168.1.1/24` → 앞 24bit가 Network, 뒤 8bit가 Host

### 예시: `192.168.1.1/24`

| 구분 | 옥텟 1 | 옥텟 2 | 옥텟 3 | 옥텟 4 |
| --- | --- | --- | --- | --- |
| **IP (10진수)** | 192 | 168 | 1 | 1 |
| **IP (2진수)** | `11000000` | `10101000` | `00000001` | `00000001` |
| **Subnet Mask (2진수)** | `11111111` | `11111111` | `11111111` | `00000000` |
| **Subnet Mask (10진수)** | 255 | 255 | 255 | 0 |

→ **Network Part**: `192.168.1.0`
→ **Host Part**: `0.0.0.1`
→ **Broadcast**: `192.168.1.255`

### 자주 쓰는 CIDR

| CIDR | Subnet Mask | 호스트 수 (사용 가능) |
| --- | --- | --- |
| `/30` | 255.255.255.252 | 2 |
| `/29` | 255.255.255.248 | 6 |
| `/28` | 255.255.255.240 | 14 |
| `/27` | 255.255.255.224 | 30 |
| `/26` | 255.255.255.192 | 62 |
| `/25` | 255.255.255.128 | 126 |
| `/24` | 255.255.255.0 | 254 |
| `/23` | 255.255.254.0 | 510 |
| `/16` | 255.255.0.0 | 65,534 |
| `/8` | 255.0.0.0 | 16,777,214 |

## 7. 공인 IP와 사설 IP

### 7.1 Public IP (공인 IP)

- 인터넷에서 동작하는 **전세계적으로 고유한** IP
- ISP가 할당, 국가별·국제 기구(IANA, APNIC 등)가 관리

### 7.2 Private IP (사설 IP)

- 인터넷에서 사용되지 않는 **내부망 전용** IP
- 조직 간 통신이 없으면 조직마다 **중복 사용 가능**
- **외부 직접 통신 불가** → NAT 필요

| 대역 | CIDR | 범위 |
| --- | --- | --- |
| Class A 사설 | `10.0.0.0/8` | `10.0.0.0` ~ `10.255.255.255` |
| Class B 사설 | `172.16.0.0/12` | `172.16.0.0` ~ `172.31.255.255` |
| Class C 사설 | `192.168.0.0/16` | `192.168.0.0` ~ `192.168.255.255` |

> **⚠️ 주의 — 사설 IP 대역 표기**
> Class B 사설 대역은 **`172.16.0.0/12`** 다 (172.16 ~ 172.31). `/16`은 오류.

## 8. NAT (Network Address Translation)

> IPv4 주소 부족 문제를 해결하기 위한 핵심 기술.
> 사설 IP를 공인 IP로 변환하여 외부와 통신.

### 8.1 NAT 종류

| 종류 | 동작 | 용도 |
| --- | --- | --- |
| **Static NAT** | 사설 IP 1개 ↔ 공인 IP 1개 (1:1 고정 매핑) | 내부 서버를 외부에 공개 |
| **Dynamic NAT** | 사설 IP ↔ 공인 IP 풀에서 동적 매핑 (1:1) | 공인 IP 풀이 있을 때 |
| **PAT (NAPT)** | 다수 사설 IP ↔ 공인 IP 1개 (포트로 구분, N:1) | **가정·회사 라우터의 기본 동작** |

### 8.2 PAT 예시

```
[내부]                  [라우터/NAT]                       [외부]
192.168.0.10:54321  →   203.0.113.5:60001        →     8.8.8.8:443
192.168.0.11:54321  →   203.0.113.5:60002        →     8.8.8.8:443
192.168.0.12:33000  →   203.0.113.5:60003        →     93.184.216.34:80
```

→ 다수의 내부 호스트가 단 하나의 공인 IP로 외부와 동시에 통신할 수 있는 이유.

## 9. IPv6 개요

- 주소 길이 **128bit** (IPv4의 4배)
- 표기: 16bit 8그룹, `:`로 구분, 16진수 표기
  - 예: `2001:0db8:85a3:0000:0000:8a2e:0370:7334`
  - 축약 가능: `2001:db8:85a3::8a2e:370:7334`
- 주소 수: 2¹²⁸ ≈ 3.4 × 10³⁸개
- NAT 불필요, IPSec 내장, Header 간소화
